Weak wording vs. strong wording

    
Both constraints "exist." Click Simulate on each to see why

❌ "Try to only use the document"

        
CONSTRAINTS:
- Try to only use the policy document.
- Let the user know if something
  isn't covered.

        
        

          🚨 Leaks or wobbles. "Try to" reads as a soft preference, not a hard rule —
          across repeated attempts the model sometimes answers directly, and even its refusals are
          phrased a different way each time, which no automated check can verify.
        

      


      

        
✅ "Answer ONLY... respond with exactly:"

        
CONSTRAINTS:
- Answer ONLY from the policy
  document provided.
- If not covered, respond with
  EXACTLY: "This topic is not
  covered in the current policy
  document. Please contact
  creditpolicy@apexbank.in"

        
        

          ✅ Holds, every time. "ONLY" plus an exact quoted fallback string leaves no
          room to interpret the rule as optional, and the literal wording means the same check can
          verify it call after call.
