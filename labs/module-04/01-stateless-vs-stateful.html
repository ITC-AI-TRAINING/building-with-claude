Stateless vs. Stateful

Building with Claude — the API has no memory; your client does

Why this matters: Claude never remembers a prior call on the server. Click Simulate below to see the same final question answered two different ways, depending only on what the client chooses to send. Fully offline — nothing sends a real request.
1

Same question, two clients

Both sides received the same three turns. Only the THIRD call differs in what gets sent.

❌ Send only the latest message
response = client.messages.create(
    ...,
    messages=[{"role": "user",
               "content": "What was my name again?"}]
)
🚨 "I don't have any information about your name or loan type on file yet — could you remind me?"
Correct behavior for what it was actually sent: this call has no earlier turns in it at all.
✅ Resend the full history
response = client.messages.create(
    ...,
    messages=self.messages   # all 5 prior turns + this one
)
✅ "Your name is Priya Sharma, and you asked about a home loan."
Correct, because "Priya Sharma" and "home loan" are still sitting in the message list this call actually sent.
