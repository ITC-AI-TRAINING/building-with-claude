MODULE 6

Grounded Answer and Citations

Building with Claude — Retrieval-Grounded Responses

Why this matters: Retrieval alone produces candidate context, not an answer — grounding is the system-prompt discipline that forces Claude to answer only from that context and show which chunk each claim came from. Everything below is fully offline — nothing sends a real request.
1

Same question, with and without grounding

Click Simulate on each to see what changes when the constraints from day4/rag_assistant.py's SYSTEM_PROMPT are removed.

Question: "What is Apex Bank's minimum credit score for a personal loan?"
❌ No grounding constraints
system = "You are a helpful
  banking assistant."

messages = [{"role": "user",
  "content": question}]
# no CONTEXT, no citation rule,
# no fallback instruction
🚨 Confident, uncited, and made up. "Apex Bank typically requires a minimum credit score of around 650 for personal loans." Sounds plausible — but no chunk says this, there's no [tag] to check, and the SOP's actual answer depends on applicant type (680 salaried / 700 self-employed / 680 government), which this answer flattens into a single guessed number.
✅ With grounding constraints
system = SYSTEM_PROMPT  # "Answer
  ONLY from CONTEXT... cite the
  source tag... fall back if the
  context doesn't contain the
  answer..."

messages = [{"role": "user",
  "content": f"CONTEXT:\n{context}
  \n\nQUESTION: {question}"}]
✅ Cited and traceable. "The Loan Processing SOP doesn't define a 'personal loan' category specifically — it sets minimums by applicant type: 680 for salaried applicants, 700 for self-employed, and 680 for government employees [sop-0]." Every number traces to a real chunk you can go open and check.
2

The retrieval check runs BEFORE Claude sees the question

Two questions, same assistant. Click Simulate on each to see whether answer_question() ever reaches client.messages.create().

✅ In-scope question
query = "What is the maximum LTV
  for a home loan?"

hits[0]["distance"] = 0.091
retrieval_check(hits)
  # 0.091 <= 0.45  ->  True
✅ Gate passes — 1 Claude call made. The Credit Policy chunk about LTV is close enough to trust; answer_question() builds the context and calls client.messages.create(), producing the cited answer from Section 1.
❌ Out-of-scope question
query = "What is Apex Bank's
  stance on cryptocurrency
  lending?"

hits[0]["distance"] = 0.81
retrieval_check(hits)
  # 0.81 <= 0.45  ->  False
🚫 Gate fails — 0 Claude calls made. Nothing in the SOP or Credit Policy is close to this topic; answer_question() returns the fixed NO_CONTEXT_FALLBACK string immediately — no API cost spent on a question the documents can't answer anyway.
