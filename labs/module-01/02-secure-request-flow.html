MODULE 1

Secure Request Flow

Building with Claude — SDK setup, key handling & request/response anatomy

Why this matters: A working Claude call is really seven small, ordered steps — load the key, verify it, build a client, send a request, get a response, and read it safely. Miss a step (or hardcode the key) and you either crash, leak a secret, or trust bad output. Everything below is fully offline and interactive — nothing sends a real request.

Hardcoded key vs. environment variable

    
Both snippets "work." Click Simulate: push to GitHub on each
      to see what actually happens to your secret the moment the code leaves your machine.


    

      

        
❌ Hardcoded key

        
client = anthropic.Anthropic(
    api_key="sk-ant-api03-7hJ2...pQ9x"
)

        
        

          🚨 Key exposed. The literal string is now permanently part of your git
          history — even if you delete the line in a later commit, it's still recoverable from the
          repo's past commits. Public repos are scanned by automated secret-scanners within
          minutes. Treat this key as compromised — revoke it in the Console immediately.
          
+ api_key="sk-ant-api03-7hJ2...pQ9x"

        

      


      

        
✅ Environment variable

        
load_dotenv()
client = anthropic.Anthropic()
# reads ANTHROPIC_API_KEY from os.environ

        
        

          ✅ Nothing sensitive pushed. .env matches a pattern in
          .gitignore, so git refuses to track it. The key never enters source control.
          
$ git status
  .env (ignored)
