MODULE 1

Tokenization & Next-Token Generation

Building with Claude — GenAI/LLM Primer

Why this matters: Every Claude API call boils down to the same loop — text is split into tokens, and the model predicts the next token one at a time until it decides to stop. Pricing, context limits, and the stop_reason field you check in every lab all come directly from this loop. Play with both interactions below; nothing here calls a real API — it's a simplified, offline model built to make the mechanics visible.
1

Tokenizer playground

Type anything and watch it split into tokens — the same unit the Claude API bills and limits you on. This is a simplified teaching tokenizer (word/punctuation splitting with long-word subword breaks), not Claude's real tokenizer — but the intuition holds: short common words ≈ 1 token, long or unusual words ≈ 2+ tokens.

Notice: debt-to-income and other long/hyphenated words split into several chips, while short words like is or a stay whole. This is why a document full of technical jargon or code often costs more tokens than plain prose of the same character length.

Next-token generation loop

Pick a starting prompt (or write your own), then click Generate next token to step through the loop one prediction at a time: candidate tokens are ranked by probability, the model picks one, appends it, and checks whether to stop.
